# Tag Browser & Manager - Complete Design

## Executive Summary

A unified interface for comprehensive tag and image management combining three core workflows:

1. **Tag Management** - Browse, categorize, rename, merge, delete tags
2. **Image Workspace** - Build image groups for bulk tag operations
3. **Statistics** - Analytics and insights about tags and categorization

---

## Core Concept: The Working Set

The key innovation is the **Working Set** - a temporary collection of images you're actively working with. Think of it like a clipboard or staging area.

## API Endpoints Required

### Tag Management

```
GET  /api/tags/browse
     ?search=<query>
     &status=all|uncategorized|needs_extended|orphaned|low_usage
     &base_category=<category>
     &extended_category=<category>
     &sort=count_desc|count_asc|alpha_asc|alpha_desc
     &offset=0
     &limit=50

GET  /api/tags/<tag_name>/detail
     Returns: tag info, sample images, implications, aliases

POST /api/tags/rename
     Body: { old_name, new_name, create_alias }

POST /api/tags/merge
     Body: { source_tags: [], target_tag, create_aliases }

POST /api/tags/delete
     Body: { tag_names: [], remove_from_images }

POST /api/tags/bulk_categorize
     Body: { tag_names: [], base_category, extended_category }

GET  /api/tags/stats
     Returns: comprehensive statistics
```

### Image Working Set Operations

```
POST /api/images/bulk_add_tags
     Body: { filepaths: [], tags: [] }

POST /api/images/bulk_remove_tags
     Body: { filepaths: [], tags: [] }

GET  /api/images/common_tags
     ?filepaths=<comma-separated>
     Returns: { all: [], some: [{tag, count, percentage}] }
```

## Technical Notes

### State Management

```javascript
// Working Set stored in localStorage for persistence
const workingSet = {
  images: ['filepath1', 'filepath2', ...],
  source: 'search:blue_hair 1girl' | 'pool:favorites' | 'manual',
  createdAt: timestamp
};

// UI State
const uiState = {
  mode: 'tags' | 'images' | 'stats',
  tagFilters: { ... },
  selectedTags: Set<string>,
  selectedImages: Set<string>,
  detailPanelOpen: boolean,
  currentTag: string | null
};
```

### Performance Considerations

1. **Virtual scrolling** for large tag/image lists
2. **Debounced search** (300ms delay)
3. **Batch API calls** for bulk operations
4. **Optimistic UI updates** with rollback on error
5. **Image thumbnail caching** via service worker
6. **Pagination** with infinite scroll option
